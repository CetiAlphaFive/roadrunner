## Algorithm вЂ” gaussian-only core MARS

### Forward pass (`mars_fit_cpp` in `src/ares.cpp`)

1. Initialize basis B with intercept column. Fit OLS, compute residual r = y в€’ fitted.
2. Per forward step:
   1. Build orthonormal basis Q of current B via modified GramвЂ“Schmidt
      (`build_Q` helper, in-place, returns rank Mq).
   2. Enumerate (parent, variable) pairs respecting interaction-degree cap.
   3. Dispatch `RcppParallel::parallelFor(0, npairs, worker)`. Each worker scans
      eligible knots for its assigned (parent, var) slice and computes the joint
      RSS reduction of the hinge pair via GramвЂ“Schmidt residual projection
      against Q.
   4. Reduce: pick the (parent, var, knot) with maximum reduction. Tie-break by
      smaller variable, then smaller knot, then smaller parent.
   5. Add the hinge pair (s = +1, then s = в€’1) as new basis columns.
   6. Refit OLS via in-place Householder QR (hand-rolled in `ols_qr`); update
      RSS and residual.
   7. If relative RSS improvement < `thresh`, stop.

### Backward pass

- Sequential subset selection. At each step, drop the term whose removal
  causes the smallest RSS increase (subject to keeping the intercept).
- Compute GCV at each subset size: `RSS / (n * (1 - (M + penalty*(M-1)/2)/n)^2)`.
- Select the subset minimizing GCV.
- Refit OLS on selected basis to obtain final coefficients.

### Parallelism (`ForwardWorker` in `src/ares.cpp`)

- Uses RcppParallel TBB (`parallelFor`).
- Worker signature takes raw `const double*` pointers вЂ” no `Rcpp::*` API calls.
- Each worker writes only to its own slot in `out` (no false sharing).
- When `nthreads <= 1`, the dispatch site short-circuits to a serial loop,
  bypassing TBB scheduler overhead.

### Numerical strategy

- All arithmetic in `double`.
- QR is hand-rolled (Householder) вЂ” no LAPACK dependency, eliminates BLAS/LAPACK
  linker complexity.
- Rank guards: candidate columns whose perpendicular norm is below
  `1e3 * eps * scale` are skipped.
- Residual is computed from full OLS each step (not residual-update tricks);
  this trades speed for numerical robustness in v0.0.0.9000.

## R surface

- `ares()` вЂ” generic with `formula` and `default` methods.
- `predict.ares()` вЂ” accepts data.frame or matrix; uses `mars_basis_cpp` to
  build the basis on `newdata`, then multiplies by `coefficients`.
- `print.ares()`, `summary.ares()`, `print.summary.ares()`, `plot.ares()` вЂ”
  standard S3 methods.

## Thread-safety contract

- The C++ entry `mars_fit_cpp` calls `RcppParallel::parallelFor` only after
  flattening Rcpp objects to `std::vector<double>` and `std::vector<int>`.
- Workers read those flattened buffers via `const double*` / `const int*`
  pointers and write only to a per-pair output slot.
- `RcppParallel::setThreadOptions(numThreads = ...)` is called from R-side
  `ares.default()` before entering C++; the C++ code observes the count via
  the `nthreads` parameter to choose serial vs parallel paths.

## Performance characteristics (v0.0.0.9001)

- Forward pass dominates runtime.
- Per-step cost: O(n Г— MВІ) for Q construction + O(npairs Г— (n + K) Г— M_q) for
  knot scoring (Friedman fast-LS prefix sums), where K = #eligible knots per
  pair (typically ~50), M_q = rank of Q (typically в‰¤ M).
- For Friedman-1 n=1500 deg=2: ares-1t ~1.1s (was ~22s in v0.0.0.9000),
  earth ~0.05s. Median single-thread ratio vs earth across the
  inst/sims grid: ~28Г— (was ~152Г—).
- The remaining wall-clock gap is now dominated by serial parts (build_Q,
  ols_qr) rather than the per-knot inner loop. Parallel scaling has shrunk
  for the same reason and is a **v0.2 roadmap item**.

## Design decisions worth knowing

- **No BLAS/LAPACK linkage**: the hand-rolled QR is sufficient at the M в‰¤ 200
  basis-term scale typical of MARS. This also dodges system-config issues
  (the dev machine had a broken MKL config that wouldn't link).
- **Selected.terms is 1-indexed**: returned to R as 1-based indices for
  consistency with `earth`'s convention.
- **Term labels**: `(Intercept)`, `h(x-cut)`, `h(cut-x)`, products joined by
  ` * ` for interactions вЂ” matches earth's labeling.
- **Drop constant columns**: emit a single `warning()`, drop, continue.
- **No global state**: `.onLoad` is a no-op. Thread count is set per-call via
  `setThreadOptions`. No package-level options.

## Testing strategy

- `tests/testthat/` вЂ” 17 `test_that` blocks, 38 expectations.
- Categories: structure, predict identity, formula method, edge cases (constant
  column, single predictor, degree=2 monotonicity, input validation),
  parity vs earth (Friedman-1 + mtcars), threading determinism, print/summary
  smoke, speed scaling.
- `inst/sims/run_mc.R` вЂ” separate Monte Carlo grid (24 cells Г— 5 reps) for
  numerical-parity benchmarking. Not exercised by `R CMD check`.

## CRAN compliance

- DESCRIPTION software names quoted: 'Rcpp', 'RcppParallel', 'earth'.
- Title in Title Case.
- Authors@R format.
- No `T`/`F`, no global `set.seed()`, no bare `print()`/`cat()` outside print
  methods, no `installed.packages()`, no `<<-`, no `.GlobalEnv` writes.
- Examples and vignettes capped at 2 threads.
- `inst/sims/` is not auto-exercised.

## What is NOT implemented (deliberate scope)

- glm responses, multi-response y, observation weights.
- `pmethod в€€ {"exhaustive", "forward", "seqrep", "cv"}`.
- `fast.k` / `fast.beta` heuristics.
- ANOVA decomposition / `evimp` variable importance.
- `linpreds` / `Auto.linpreds` (linear-only directions are reserved as
  `dirs == 2` but not currently produced by the forward pass).
